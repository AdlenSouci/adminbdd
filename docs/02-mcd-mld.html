MCD → MLD

  • Entité → table (sauf fusion 1-1 choisie).
  • Attribut → colonne (formes normales).
  • Relation → FK + contraintes.

Par cardinalité max

CasRègle
1-1Fusion ou PK entité forte = FK (+ PK) entité faible
1-NLa PK de l’entité dont la card. max = N (0,N ou 1,N) devient une FK dans l’entité dont la card. max = 1 (0,1 ou 1,1) — comme dans ton PDF « Du MCD au MLD ».
N-NTable pivot : PK souvent (FK1, FK2)

Exemples

User–Profile 1-1 : fusion ou double table liée.
Pays–Ville 1-N : ville.code_pays FK.
Acteur–Film N-N : table jouer.

Outils

Mocodo · Looping (MCD + SQL).
