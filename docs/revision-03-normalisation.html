Normalisation

Formes normales et décompositions — d’après le cours PDF « Normalisation des bases de données ».

Objectifs du cours

  • Connaître 1NF, 2NF, 3NF, BCNF.
  • Repérer les anomalies d’un schéma mal conçu.
  • Savoir décomposer pour cohérence et moins de redondance.

Pourquoi normaliser ?

  • Redondance : même fait répété sur plusieurs lignes.
  • Incohérences : deux valeurs contradictoires pour un même fait.
  • Anomalies de mise à jour : insertion (donnée partielle impossible), modification (trop de lignes à toucher), suppression (perte d’info utile).
  • But : relations structurées pour limiter ces effets.

Exemple de départ (problème)

Lignes type : Étudiant, Cours, Professeur, Département (Alice/BD/Dupont/Info…).

  • Le département « Informatique » est répété.
  • Changement de nom de dept = risque d’oubli sur certaines lignes.
  • Risque d’incohérence entre professeur et département affiché.

1NF — première forme normale

  • Toute valeur d’attribut est atomique (indivisible).
  • Pas de groupe répété dans une même cellule.
  • Contre-exemple : colonne Cours = « BD, Réseaux » dans une case → viole la 1NF.
  • Conforme : une ligne par couple (étudiant, cours) avec une valeur par colonne.

2NF — deuxième forme normale

  • Être en 1NF.
  • Chaque attribut non-clé dépend de la clé entière, pas d’une partie seulement de la clé composée.

Exemple cours

  • Clé (Étudiant, Cours) : VilleÉtudiant ne dépend que de Étudiant → dépendance partielle → pas 2NF.

Décomposition 2NF (idée)

  • Table Étudiant : Étudiant, VilleÉtudiant.
  • Table Cours : Cours, Professeur, Département.
  • Table Inscription : Étudiant, Cours (liaison).

3NF — troisième forme normale

  • Être en 2NF.
  • Aucun attribut non-clé ne dépend transitivement de la clé via un autre non-clé.
  • En clair : tout non-clé dépend directement de la clé.

Exemple cours

  • Table Cours, Professeur, Département avec Cours → Professeur et Professeur → Département.
  • Donc Cours → Département en transitif → pas 3NF.

Décomposition 3NF

  • Cours : Cours, Professeur.
  • Professeur : Professeur, Département.

BCNF — Boyce-Codd

  • Pour chaque DF X → Y, X est une clé candidate.
  • Version plus stricte que la 3NF (certains cas 3NF mais pas BCNF existent en théorie).
