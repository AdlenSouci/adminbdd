GITHUB PAGES — mettre les fiches en ligne (gratuit)
====================================================

1) Cree un depot GitHub.

2) Pour que Pages marche : le dossier "docs/" a la racine du repo avec au
   minimum index.html, styles.css, pdf-source-*.html (pages = texte extrait
   des PDF). Le reste du projet sur ton PC n'est pas obligatoire sur GitHub.

   Exemple minimal :
     README.md     (optionnel)
     docs/
       index.html
       styles.css
       pdf-source-01-mcd.html ... pdf-source-08-droits.html
       _build_sources_from_pdf_extracts.py (optionnel)

3) Sur GitHub : Settings (du depot) > Pages

4) Source : Deploy from a branch
   Branch : main (ou master)
   Folder : /docs
   Save

5) Attends 1–2 minutes. L’URL sera du type :
   https://TON_USER.github.io/NOM_DU_REPO/
   (GitHub affiche l’URL exacte dans la section Pages.)

6) Sur ton telephone : ouvre cette URL dans Chrome / Safari. Ajoute aux favoris.

Notes
-----
- Si le site affiche 404 : verifie que "docs" est bien a la racine du depot et que index.html est dedans.
- Tu peux mettre a jour les .html et faire git push ; le site se met a jour tout seul.
- Pas besoin de Jekyll : fichiers HTML statiques suffisent.

Alternative sans GitHub
-----------------------
Ouvre docs/index.html en local (double-clic) : ca marche sur ton PC.
Sur telephone : envoie le dossier docs en zip sur Drive et ouvre index.html avec un navigateur (selon appareil, chemins relatifs vers styles.css peuvent poser souci) — GitHub Pages evite ce probleme.
