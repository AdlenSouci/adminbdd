04 — DDL

Structure SQL

Commandes

CREATE · ALTER · DROP · TRUNCATE → structure (≠ DML données).

Types (rappel)

Num : INT, BIGINT, DECIMAL… · Date : DATE, TIME, TIMESTAMP, DATETIME (MySQL)… · Texte : CHAR, VARCHAR, TEXT… · BOOL, BLOB, JSON, UUID (selon SGBD).

Clé primaire

Unique · NOT NULL · une par table · simple ou composée · CONSTRAINT pk_….

Auto-incrément

SGBDSyntaxe
MySQLAUTO_INCREMENT
PostgreSQLSERIAL (vieux) ou GENERATED ALWAYS/BY DEFAULT AS IDENTITY ⭐
SQL ServerIDENTITY(seed, inc)
Oracle 12c+GENERATED AS IDENTITY · <12c : SEQUENCE+TRIGGER
SQLiteINTEGER PRIMARY KEY / AUTOINCREMENT

PG ALWAYS = pas d’insert manuel sur id · BY DEFAULT = imports possibles.

Clés étrangères

REFERENCES · ON DELETE / ON UPDATE : CASCADE, SET NULL, SET DEFAULT, RESTRICT / NO ACTION.
FK composée alignée sur PK parent.
CASCADE si l’enfant n’a pas de sens sans le parent · RESTRICT pour référence métier (client).

Contraintes

PK · FK · UNIQUE · NOT NULL · CHECK · DEFAULT. Nommer : pk_ fk_ uk_ chk_.

ALTER / DROP / TRUNCATE

ALTER TABLE · DROP TABLE [IF EXISTS] [CASCADE] · TRUNCATE rapide · PG RESTART IDENTITY · CASCADE tables liées.

Bon réflexe

Contraintes nommées · FK avec stratégie delete/update · NOT NULL + CHECK · migrations (Flyway, Liquibase, Alembic…).

PK vs UNIQUE

PK = une par table · jamais NULL. UNIQUE = plusieurs possibles · NULL souvent OK.

CASCADE quand ?

Oui : composition (commande→lignes). Non : référence (commande→client) → RESTRICT ou SET NULL.
