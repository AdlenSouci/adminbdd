Données ≠ traitements · démarche itérative.

Entité

Objet réel identifiable, occurrences homogènes. Nom au singulier, vocabulaire métier, pas d’abréviation obscure (CLIENT ✓ / CLI ✗).

Attributs

  • Simple : indivisible (piste 1NF).
  • Calculé : dérivé d’autres (ex. âge).
  • Attention composition (ex. n° sécu).

Associations

Binaire · Ternaire (3 entités) · N-aire · Réflexive (ex. superviser).

Cardinalités min,max

0 = optionnel · 1 = obligatoire · N = plusieurs. Types : 0,1 · 1,1 · 0,N · 1,N.

→ MLD : côté (0,1) ou (1,1) → souvent FK migrée vers l’entité « N » (ex. voiture → personne).

Identifiant

Unicité · NOT NULL · stabilité · minimalité. Mauvais : nom+prénom, email seul, téléphone. Souvent id technique + attribut métier (code-barres) en colonne.

Clé candidate / primaire

Plusieurs candidates possibles · une PK choisie (#) : stabilité, simplicité, perf, sens métier.

Ex. bibliothèque (cours)

LIVRE · AUTEUR · ADHÉRENT · ECRIRE (N,N) · EMPRUNTER (N,N + dates). EXEMPLAIRE (1,N livre) · CATÉGORIE (N,N livre).

Check révision

Identifiants stables · cardinalités cohérentes · pas de redondance · entité vs attribut (adresse → entité ?)

Pièges

Entité confondue avec attribut · mauvaise lecture des cardinalités · sur-identification.
