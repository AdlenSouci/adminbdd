DDL avancé & optimisation des schémas

Séance type DBA : partitions, index avancés, OLTP vs OLAP (support cours).

Objectifs

  • DDL avancé : partitions, index complexes, contraintes au bon niveau.
  • Adapter le schéma aux charges OLTP et OLAP.

Trois piliers (cours)

  • Partitionnement : très grosses tables, maintenance ciblée.
  • Index complexes : composites, couverture, partiels (selon SGBD).
  • Choix OLTP / OLAP : normalisation vs dénormalisation, modèles analytiques.

Partitionnement par plage (RANGE)

  • Idéal données temporelles (logs, commandes par année/mois).
  • La colonne de partition doit souvent faire partie de la clé primaire (ex. (id, sale_date)).
  • Le moteur fait du partition pruning : ne lit que les partitions utiles à la requête.
  • Ex. syntaxe type MySQL : PARTITION BY RANGE (YEAR(sale_date)) avec VALUES LESS THAN.

Partitionnement par liste (LIST)

  • Données catégorielles : région, pays, statut.
  • Ex. : partitions nord / sud par valeurs de region.

Partitionnement par hachage (HASH)

  • Répartit les lignes sur N partitions de façon uniforme.
  • Utile pour répartir l’écriture et éviter un « point chaud » quand il n’y a pas de colonne de partition logique évidente.

Index composite

  • Plusieurs colonnes dans un seul index : l’ordre compte.
  • Règle d’or cours : mettre en premier la colonne la plus sélective (qui filtre le plus).
  • Ex. mauvais : (status, department_id) si status peu discriminant ; bon : (department_id, status).

Index de couverture (covering)

  • L’index contient toutes les colonnes lues par la requête (filtre + projections).
  • Le moteur peut répondre sans retourner à la table heap (très efficace en lecture).
  • Ex. requête sur username retournant email, last_login → index (username, email, last_login).

OLTP vs OLAP

OLTPOLAP
UsageTransactions du quotidien (e-commerce, banque)Analyse, entrepôt
ChargeNombreuses petites écrituresGrosses lectures, agrégations
SchémaNormalisé (ex. 3NF)Souvent dénormalisé, étoile / flocon
IndexCiblés, peu de colonnesLarges, couverture fréquente
  • Image cours : OLTP = sprinter ; OLAP = marathonien — on ne dimensionne pas pareil.

Atelier (rappel)

  • Partitionner une table de commandes par date.
  • Index composites pour rapports et recherches.
  • Justifier chaque choix (charge, requêtes types).
