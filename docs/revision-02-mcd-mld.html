Du MCD au MLD

Méthodologie MERISE : analyse → conception → réalisation (d’après le support « Du MCD au MLD »).

MERISE en trois temps

  • Analyse : comprendre le domaine ; entités, attributs, relations → MCD.
  • Conception : MCD → MLD (normalisation, règles de passage).
  • Réalisation : MLD → schéma relationnel implémenté (SQL, appli).
  • Données et traitements sont modélisés séparément pour clarifier et faciliter la maintenance.

Vue d’ensemble du processus

  • Processus itératif : chaque étape a des objectifs et des livrables.
  • Le MCD est visuel (rectangles = entités, traits = relations), indépendant du technique.
  • Le MLD est une vue logique normalisée : tables, colonnes, intégrité référentielle.

Étape 1 — Analyse conceptuelle

  • Recueil des besoins : entretiens, questionnaires, analyse de documents.
  • Entités + attributs ex. université : Étudiant (matricule, nom…), Cours (code, titre…), Professeur…
  • Relations : « un étudiant suit un cours », « un prof enseigne un cours », etc.
  • Construction du MCD comme base de la conception logique.

Outils de modélisation (cours)

  • MOCODO.net : MERISE, MCD, génération SQL.
  • Looping : MCD en ligne + SQL.

Étape 2 — Conception logique

  • Transformer le MCD en MLD avec normalisation et règles de transformation.
  • Entité MCD → table MLD.
  • Attributs répartis dans les tables selon les formes normales.
  • Relations → contraintes d’intégrité référentielle (clés étrangères).

Règle 1 — Toute entité → une table

  • Chaque entité du MCD devient une table dans le MLD (schéma relationnel).

Cas 1:1 (max 1 des deux côtés)

  • Ex. user (0,1) — profile (1,1) avec association has.
  • Option A : fusionner en une seule table si tout est systématiquement lié 1:1.
  • Option B : deux tables : la clé primaire de l’entité « forte » devient FK + PK de l’entité faible (profil référence user).

Cas 1:N (un côté max = N, l’autre max = 1)

  • Ex. pays (1,N) — ville (1,1) via possede.
  • La clé primaire de l’entité côté N est recopiée en clé étrangère dans l’entité côté 1 (ville référence le pays).
  • (Aligné sur la formulation du support : PK du côté cardinalité max N → FK dans l’entité max 1.)

Cas N:N (max N des deux côtés)

  • Ex. acteur (1,N) — film (1,N) via jouer.
  • L’association devient une table de liaison (pivot / table d’association).
  • Clé primaire souvent composée des deux FK vers les deux tables.
