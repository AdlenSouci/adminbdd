07 — EXPLAIN

MySQL & PostgreSQL

Texte intégral du fichier cours (explain.txt)

À quoi ça sert

Voir le plan : scans, index, jointures → optimiser les requêtes lentes.

MySQL — commandes

EXPLAIN SELECT … · EXPLAIN EXTENDED + SHOW WARNINGS · EXPLAIN FORMAT=JSON

MySQL — lire le plan

  • type (pire → mieux) : ALL → index → range → ref → eq_ref → const / system
  • key : index utilisé (NULL = problème)
  • rows : estimation
  • Extra : Using where · Using index · Using temporary · Using filesort · Using index condition
  • select_type : SIMPLE, PRIMARY, SUBQUERY, DERIVED, UNION…

MySQL — action type

Full scan sur gros filtre → index adapté + ANALYZE TABLE si stats vieilles.

PostgreSQL

EXPLAIN (ANALYZE, BUFFERS) SELECT …
ANALYZE = temps réels · BUFFERS = cache vs disque.
Nœuds : Seq Scan · Index Scan · Bitmap Heap Scan · Hash Join · Nested Loop · Parallel…
Lire : actual time · lignes réelles vs estimées · Rows Removed by Filter · shared hit / read.

MySQL vs PG

PG : mesures réelles natives. MySQL : plan surtout estimé. MySQL : FORMAT=JSON natif.
