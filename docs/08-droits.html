08 — Droits & utilisateurs

MySQL · DCL · rôles

Texte intégral du fichier cours (gestion_droit.txt)

Rappel

Compte MySQL ≠ utilisateur appli (ligne en table). L’appli utilise un user BDD dédié (éviter root vide en prod).

Lister les comptes

SELECT user, host FROM mysql.user; (détails selon version / plugin d’auth)

Niveaux de privilèges

Global (*.*) · Base (db.*) · Table · Colonne · Routines

Comptes

CREATE USER 'u'@'h' IDENTIFIED BY '…'; · ALTER USER · DROP USER
Hôte : localhost, IP, %, sous-réseau, sous-domaine. Même nom + hôte différent = 2 comptes.
unix_socket : auth système (ex. www-data) — à manier avec prudence.

GRANT / REVOKE

GRANT SELECT ON db.t TO 'u'@'h'; · REVOKE … FROM …
WITH GRANT OPTION : peut transmettre les mêmes droits reçus.
Révocation : pas de cascade sur une chaîne user1→user2→user3. Effet session possible.
Droits par colonne : GRANT SELECT, INSERT(col) ON …
SHOW GRANTS FOR 'u'@'localhost';

Rôles (MySQL 8+)

CREATE ROLE · GRANT … TO role · GRANT 'role' TO 'u'@'h'
Rôles imbriqués : GRANT 'r1' TO 'r2'
SET ROLE · SET DEFAULT ROLE … FOR … · REVOKE role FROM · DROP ROLE

Exo emptemp (rappel)

Rôles admin / analyste / gestionnaire · utilisateurs · SET DEFAULT ROLE · tests mysql -u …
