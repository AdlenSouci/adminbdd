OUVRIR LES FICHES AU BON ENDROIT
================================

1) Chemin EXACT sur ton PC (explorateur Windows, copier-coller dans la barre d’adresse) :

   C:\Users\adlen\Downloads\adminbdd\docs\index.html

   Clic droit > Ouvrir avec > Chrome, Edge ou Firefox.

2) Comment savoir que c’est la bonne version ?

   En haut de l’accueil tu dois lire : « Build 15 » (pas « build 12 », pas de bandeau jaune ancien).

   Si ce n’est pas le cas : Ctrl + Shift + R  (ou fenêtre de navigation privée).

3) Les pages « Texte brut » (pdf-source-…) restent des extraits bruts du PDF : c’est normal qu’elles soient moches.

   Les pages « revision-… » sont les fiches de révision : le style est DÉJÀ dans le fichier HTML (build 15).

4) GitHub Pages (adresse github.io)

   Ce que tu vois en ligne = ce qui est sur GitHub après git push. Ton dossier local peut être plus récent que le site.

5) Cursor peut ouvrir un autre dossier que Downloads\adminbdd

   Si les fichiers ne changent pas dans l’éditeur, vérifie dans Cursor : Fichier > Ouvrir le dossier… et choisis le dossier qui CONTIENT docs\index.html.

6) Après modification de fiche.css

   Pour recopier le CSS dans toutes les fiches :

   python docs\embed_fiche_css.py

   (à lancer depuis la racine du projet adminbdd)
