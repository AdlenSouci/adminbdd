DDL — Data Definition Language

Définir et modifier la structure des objets (pas les données : ça, c’est le DML).

Commandes principales

  • CREATE : tables, vues, index…
  • ALTER : modifier un objet existant.
  • DROP : supprimer un objet.
  • TRUNCATE : vider une table (structure conservée).

CREATE TABLE (squelette)

  • Colonnes : nom TYPE [contraintes], puis contraintes de table éventuelles.
  • Choisir des types adaptés (taille, précision) dès la conception.

Types courants

  • Numériques : INTEGER, BIGINT, SMALLINT, DECIMAL(p,s), FLOAT…
  • Date/heure : DATE, TIME, TIMESTAMP ; DATETIME (MySQL / SQL Server).
  • Texte : CHAR(n) fixe, VARCHAR(n) variable, TEXT/CLOB long.
  • Autres : BOOLEAN, BLOB, JSON (MySQL/PG), UUID (PG).

Clé primaire (PK)

  • Unicité + NOT NULL ; une seule PK par table.
  • Peut être composée : PRIMARY KEY (a, b).
  • Syntaxe : au niveau colonne, au niveau table, ou CONSTRAINT pk_… PRIMARY KEY (…).

Auto-incrémentation (rappel SGBD)

SGBDApproche
MySQL / MariaDBAUTO_INCREMENT
PostgreSQLSERIAL (ancien) ou GENERATED … AS IDENTITY (recommandé, SQL standard)
SQL ServerIDENTITY(seed, inc)
Oracle 12c+GENERATED … AS IDENTITY
Oracle < 12cSEQUENCE + trigger
SQLiteINTEGER PRIMARY KEY ; AUTOINCREMENT si pas de réutilisation d’id
  • GENERATED ALWAYS : pas d’insertion manuelle de l’id (sûr).
  • GENERATED BY DEFAULT : insertion manuelle possible (imports).

Clé étrangère (FK)

  • Lie une table enfant à une table parent ; garantit l’intégrité référentielle.
  • Syntaxe : REFERENCES parent(col) ou FOREIGN KEY … CONSTRAINT fk_….
  • FK composée possible si la PK parent est composée (ex. notes → inscriptions).

ON DELETE / ON UPDATE

  • CASCADE : propage suppression / mise à jour.
  • SET NULL / SET DEFAULT : selon colonnes nullable / défaut.
  • RESTRICT / NO ACTION : bloque si lignes enfants (comportement par défaut souvent).

Autres contraintes

ContrainteRôle
UNIQUEUnicité ; plusieurs NULL souvent autorisés (selon SGBD)
NOT NULLValeur obligatoire
CHECKCondition (ex. prix > 0, date_fin > date_debut)
DEFAULTValeur si rien fourni ; ON UPDATE CURRENT_TIMESTAMP (MySQL) pour mise à jour auto

Nommage des contraintes

  • Ex. cours : pk_table, fk_enfant_parent, uk_table_col, chk_table_desc.
  • Facilite les ALTER … DROP CONSTRAINT et la maintenance.

ALTER TABLE

  • Ajouter colonne / contrainte ; supprimer colonne ou contrainte.
  • Modifier le type : PG ALTER COLUMN … TYPE ; MySQL souvent MODIFY COLUMN.
  • Renommer la table : RENAME TO (syntaxe selon SGBD).

DROP / TRUNCATE

  • DROP TABLE : supprime table + données ; IF EXISTS ; CASCADE pour dépendances (PG).
  • TRUNCATE : vide vite ; PG RESTART IDENTITY ; CASCADE pour tables liées si besoin.

Bonnes pratiques (cours)

  • Nommer toutes les contraintes ; préférer IDENTITY standard quand possible.
  • FK avec actions ON DELETE / ON UPDATE réfléchies (CASCADE pour composition, RESTRICT pour références métier).
  • NOT NULL sur colonnes obligatoires ; CHECK pour règles métier.
  • Éviter types trop larges sans raison ; penser aux index ; sauvegarder avant grosses modifs DDL.

PK vs UNIQUE (FAQ cours)

  • Une seule PK par table, jamais NULL, identifie la ligne.
  • Plusieurs UNIQUE possibles ; NULL souvent autorisé une ou plusieurs fois.
  • En pratique : PK = identifiant principal ; UNIQUE = emails, codes alternatifs.

Quand CASCADE sur DELETE ?

  • Oui si l’enfant n’a pas de sens sans le parent (lignes de commande → commande).
  • Non si référence « libre » (commande → client) : plutôt RESTRICT ou SET NULL.
