06 — Sauvegarde & résilience

RPO · RTO · PITR

Texte intégral extrait du PDF

Pourquoi

Panne disque · erreur humaine (DELETE) · corruption · ransomware · RGPD / image / continuité.

RPO

Recovery Point Objective = volume max de données perdues acceptable. Ex. RPO 1 h → backup horaire.

RTO

Recovery Time Objective = délai max pour service rétabli. Ex. RTO 30 min → procédure + infra adaptées.

Types de backup

  • Complète : simple · lourd.
  • Incrémentielle : delta depuis dernière sauvegarde.
  • Différentielle : delta depuis dernière complète.

PITR

  1. Restaurer la dernière complète
  2. Rejouer les journaux jusqu’à l’instant voulu

MySQL : binary logs (log-bin=mysql-bin). Sans journaux → PITR impossible en pratique.

Automatisation

Nom daté · compression · alerte si échec · rotation des fichiers.

Règle 3-2-1

3 copies · 2 types de support · 1 copie hors site.

Atelier

Script auto · simuler suppression · restaurer · mesurer le RTO réel.
