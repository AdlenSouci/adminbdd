Droits, utilisateurs & rôles (MySQL / MariaDB)

Synthèse du support gestion_droit.txt (DCL / gestion des comptes).

Utilisateur BDD ≠ utilisateur appli

  • Compte MySQL = identité pour se connecter au serveur (PHP, admin, etc.).
  • Utilisateur « adhérent » dans une table = métier applicatif, autre chose.
  • L’appli utilise en général un compte technique dédié avec droits limités.
Exemple WAMP : root sans mot de passe = pratique locale, interdit en production.

Voir les comptes (MySQL / MariaDB)

  • Table système : mysql.user (colonnes selon version ; en 8+ souvent authentication_string plutôt que password).

Niveaux de privilèges

  • Global (*.*) : ex. CREATE USER, RELOAD.
  • Base (ma_base.*) : CREATE, SELECT, etc.
  • Table / colonnes / routines : granularité fine.

CREATE USER

  • CREATE USER [IF NOT EXISTS] 'nom'@'hôte' IDENTIFIED BY '…';
  • hôte : localhost, IP, nom de domaine, % (tout origine — attention).
  • Même nom + hôtes différents = comptes distincts.
  • Variantes auth : IDENTIFIED VIA unix_socket (délégation OS), LDAP, etc.
  • ALTER USER … ; DROP USER [IF EXISTS] ….

DML courants à accorder

  • SELECT, INSERT, UPDATE, DELETE — sur table ou colonnes précises.

GRANT

  • Ex. : GRANT SELECT ON test.livre TO 'user1'@'localhost';
  • WITH GRANT OPTION : l’utilisateur peut redistribuer les droits qu’il a reçus (transmission).
  • Sans droit de créer un utilisateur, on peut quand même GRANT si on a l’option sur ces privilèges.

REVOKE

  • REVOKE SELECT ON test.livre FROM 'user2'@'localhost';
  • Effet parfois visible après fin de session selon contexte (notion évoquée dans le support).
  • Pas de cascade sur la transmission : si A transmet à B et B à C, révoquer B ne retire pas automatiquement ce que C a reçu de B (logique à retenir pour l’examen).

Droits par colonne (exercice type)

  • Accorder SELECT, INSERT(title), UPDATE(title) seulement sur livre pour exclure description.
  • Alternative : GRANT … ON table puis REVOKE INSERT(description), UPDATE(description) ….

Vérifier les droits

  • SHOW GRANTS FOR 'user'@'localhost'; (attention à l’orthographe de l’hôte dans la commande).

Rôles (MySQL 8+ / MariaDB 10+)

  • CREATE ROLE 'nom_role';
  • GRANT … ON db.* TO 'nom_role'; puis GRANT 'nom_role' TO 'user'@'hôte';
  • Rôles imbriqués : GRANT 'role_a' TO 'role_b'; pour hiérarchies.
  • SET ROLE 'nom_role'; — un rôle attribué n’est actif qu’après activation.
  • SET DEFAULT ROLE 'nom_role' FOR 'user'@'hôte'; pour activation à la connexion.
  • REVOKE 'role' FROM 'user'@'hôte'; · DROP ROLE 'nom_role';

Exemple parcours (cours)

  • Créer rôles lecteur, editeur ; GRANT SELECT sur base au lecteur ; SELECT, INSERT, UPDATE à l’éditeur.
  • Assigner les rôles aux utilisateurs ; SET DEFAULT ROLE ; contrôler avec SHOW GRANTS.
